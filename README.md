# Утилита для проверки файлов конфигурации

## Описание проекта

Проект позволяет проанализировать файл конфигурации в формате JSON или YAML и обратить внимание на потенциально опасные настройки. Можно анализировать как отдельный файл, так и все файлы в конкретной директории. В случае нахождения недочётов происходит выход с ошибкой. 

### Критерии опасности настроек, проверяемые программой:

- уровень логирования
- указание пароля в открытом виде
- использование `0.0.0.0` без ограничений
- отключённый TLS
- использование устаревших или небезопасных алгоритмов
- слишком широкие права доступа, подлежащие установке
- слишком широкие права доступа файла, указанного в файле конфигурации
- слишком широкие права доступа файла конфигурации

## Запуск проекта

Запуск предполагает передачу в качестве аргумента:
- либо файла .json, .yaml, .yml;
- либо директории;
- либо (с флагом --stdin) строки в стандартном вводе, которая поддаётся парсингу как JSON или YAML
- либо без аргументов, но с флагами --api для запуска в качестве REST API-сервера и --grpc для запуска в качестве gRPC-сервера

```
./Analyze-Cfg config.json
```
*Вывод*
```
1. HIGH: Сервер слушает на всех хостах (0.0.0.0). Ограничьте доступ. Для локального доступа используйте 127.0.0.1 .

2. HIGH: Пароль в открытом виде. Используйте переменные окружения.

3. HIGH: TLS проверка выключена. Включите TLS-проверку.

4. LOW: логирование в debug-режиме. Поменяйте режим на более избирательный (info+).

5. HIGH: слишком слабый алгоритм - MD5. Замените его на более безопасный.

6. HIGH: слишком широкие права доступа. Поменяйте на менее широкие (например, 750 или 700).


Ошибка: потенциально опасные настройки обнаружены
```

## Проверка конфигурации из стандартного ввода с флагом --stdin
```
./Analyze-Cfg --stdin <<EOF
{"server": {"host": "0.0.0.0", "tls_verify": false}, "database":{"password": "123"}, "storage": {"path":"storage/storage.txt", "permissions":"0777", "digest_algorithm":"MD5"}, "log": {"output":"stdout", "level":"debug"}}
EOF
```
*Вывод*

```
Обнаружено 6 потенциально опасных настроек:

1. HIGH: Сервер слушает на всех хостах (0.0.0.0). Ограничьте доступ. Для локального доступа используйте 127.0.0.1 .

2. HIGH: Пароль в открытом виде. Используйте переменные окружения.

3. HIGH: TLS проверка выключена. Включите TLS-проверку.

4. LOW: логирование в debug-режиме. Поменяйте режим на более избирательный (info+).
```

## Запуск в качестве REST API (флаг --api)

С помощью флага --api можно запустить утилиту как REST API-сервер, принимающий массив конфигурационных настроек и возвращающий массив проблем с отдельными полями для пути к проблемному параметру, описанием проблемы, рекомендацией по устранению и степенью критичности

```
./Analyze-Cfg --api
```

```
curl -X POST http://localhost:8080/analyze \
-H "Content-Type: application/json" \
-d '{"log": {"output":"stdout", "level":"debug"}}'
```
*Вывод*
```
[{"path":"server.tls_verify","description":"TLS проверка выключена.","recommendation":"Включите TLS-проверку.","severity":"HIGH"},{"path":"log.level","description":"логирование в debug-режиме.","recommendation":"Поменяйте режим на более избирательный (info+).","severity":"LOW"}]
```
## Запуск в качестве gRPC

С помощью флага --grpc можно запустить утилиту как REST API-сервер, принимающий массив конфигурационных настроек и возвращающий массив проблем с отдельными полями для пути к проблемному параметру, описанием проблемы, рекомендацией по устранению и степенью критичности

```
./Analyze-Cfg --grpc
```

```
grpcurl -plaintext -d '{"server":{"host": "0.0.0.0"}, "log": {"output":"stdout", "level":"debug"}}' localhost:8080 analyze.CfgAnalyzer/Analyze
```

*Вывод*
```
{
  "problems": [
    {
      "path": "server.host",
      "description": "Сервер слушает на всех хостах (0.0.0.0).",
      "recommendation": "Ограничьте доступ. Для локального доступа используйте 127.0.0.1 .",
      "severity": "HIGH"
    },
    {
      "path": "log.level",
      "description": "логирование в debug-режиме.",
      "recommendation": "Поменяйте режим на более избирательный (info+).",
      "severity": "LOW"
    }
  ]
}
```
## Тихий режим с флагами -s или --silent

В тихом режиме работа завершается без ошибки в случае нахождения опасных настроек. Неприменимо к запуску в качестве REST API-сервера и в качестве gRPC-сервера

```
./Analyze-Cfg -s config.json
```

```
./Analyze-Cfg --silent config.json
```
